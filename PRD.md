# 10년 일기장 (10-Year Diary) PRD

## 1. 개요 (Overview)
* **제품명**: 10년 일기장
* **목적**: 매일의 작은 순간들을 기록하고, 같은 날짜의 과거 기록을 한눈에 보며 추억할 수 있는 서비스.
* **플랫폼**: 모바일 및 PC 대응 반응형 웹 애플리케이션
* **데이터 관리**: Firebase 연동 (여러 기기에서 실시간 동기화)

## 2. 핵심 기능 요구사항
1. **사용자 인증 (Authentication)**
   * Google 계정 기반 소셜 로그인 기능
   * 기기 변경 시에도 동일한 데이터 접근 보장
2. **10년 일기 작성 및 조회 (Daily View)**
   * 중앙 상단에 현재 선택된 날짜 (예: "4월 22일") 표시 및 네비게이션 (이전/다음 날 이동)
   * 선택된 날짜에 대해 직근 10년치(예: 2017년~2026년)의 일기 슬롯을 세로 목록으로 배치
   * 각 슬롯에서 해당 연도의 일기를 조회, 작성, 수정 가능
   * 내용이 작성된 슬롯과 빈 슬롯의 시각적 구분
3. **달력 내비게이션 (Calendar View)**
   * 월간 달력을 제공하여 원하는 날짜로 빠르게 이동 가능
   * 특정 날에 일기가 1개라도 작성되어 있다면 시각적 표시 (예: 점, 하이라이트)
4. **반응형 UI (Responsive Design)**
   * 모바일 화면에서는 세로형 스크롤에 최적화된 레이아웃 제공
   * PC 화면에서는 넓은 화면을 활용한 쾌적한 뷰 제공

## 3. 사용자 흐름 (User Flow)
1. **진입 및 로그인**: 앱 접속 -> 'Google로 로그인' 클릭
2. **오늘의 기록**: 로그인 후 오늘 날짜의 10년 일기 뷰로 자동 진입
3. **일기 작성**: 금년 또는 과거의 빈 슬롯을 클릭하여 텍스트 작성 후 저장
4. **과거 회상**: 기록된 과거 연도의 텍스트를 읽으며 추억 회상
5. **날짜 이동**: 달력에서 다른 날로 이동하여 빈 공간을 채우거나 지난 기록 감상

## 4. 데이터베이스 아키텍처 구조 (Firestore)
* **Collection**: `users`
  * **Document**: `{userId}` (인증된 사용자 ID)
    * **Subcollection**: `entries`
      * **Document**: `{MM-DD}` (예: "04-22", 연도를 제외한 '월-일' 기준)
        * **Fields**:
          * `years`: Map 구조로 각 연도를 키로 사용하여 내용 저장
            * 예: `{ "2026": "비가 옴", "2025": "바쁘게 보냄" }`
          * `updatedAt`: Firestore Timestamp
